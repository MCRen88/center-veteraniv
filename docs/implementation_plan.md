# План впровадження вебресурсу Кваліфікаційного центру

Цей документ описує план створення вебресурсу для Кваліфікаційного центру ветеранів відповідно до нормативних документів (Постанови КМУ № 986, № 956 та Професійного стандарту фахівця із супроводу ветеранів війни та демобілізованих осіб).

## Мета
Створити сучасний, функціональний та естетично привабливий вебпортал для Кваліфікаційного центру, який забезпечить публічне інформування, автоматизує подачу заяв кандидатами та дозволить ветеранам проходити тренувальне онлайн-тестування за професійним стандартом.

---

## User Review Required

> [!IMPORTANT]
> ### Вибір технологічного стеку та архітектури
> * **Фреймворк**: React + Vite + TypeScript. Це забезпечить швидке завантаження, компонентну архітектуру та типізацію.
> * **Стилізація**: Vanilla CSS + CSS Variables для розробки преміального дизайну з темною та світлою темами, плавними градієнтами, ефектом Glassmorphism (скло) та мікроанімаціями.
> * **Збереження даних**: Клієнтська база даних на основі `localStorage` для збереження стану тестування, поданих заяв та реєстру сертифікатів. Це дозволить запустити повністю робочий прототип без необхідності налаштування складних серверних баз даних на цьому етапі.

## Proposed Changes

Створення структури React-застосунку з наступними компонентами:

1. **Головна сторінка (Landing Page)**:
   * Презентація Кваліфікаційного центру з патріотичним та преміальним дизайном (глибокий синій, темно-сірий, золоті акценти, плавні градієнти).
   * Інформаційний блок про нормативно-правову базу (Постанови 986, 956, Стандарт 508).
   * Покроковий гайд: «Як отримати сертифікат фахівця».
   * Калькулятор відповідності освіти: інтерактивна форма, що перевіряє відповідність спеціальності кандидата вимогам профстандарту (9 дозволених галузей знань).

2. **Модуль онлайн-тестування (Test Simulator)**:
   * Інтерактивний тест на **24 унікальних запитання** (по 3 запитання на кожну з 8 трудових функцій від А до Ж).
   * Питання розроблені на основі реальних знань і кейсів з професійного стандарту.
   * Анімована шкала прогресу, можливість перегляду пояснень після відповіді (у режимі навчання).
   * Екран результатів:
     * Аналітика відповідей у розрізі трудових функцій (діаграма або візуальні шкали відсотків).
     * Рекомендації: які теми/функції потрібно підтягнути.
     * Можливість згенерувати та завантажити Mock-сертифікат у разі успішного складання (>75%).

3. **Майстер подачі заяв (Application Wizard)**:
   * Покрокова онлайн-форма подачі документів (згідно з Постановою 956):
     * Крок 1: Персональні дані та контакти.
     * Крок 2: Відомості про освіту та досвід.
     * Крок 3: Завантаження сканованих документів (Drag-and-drop зона).
     * Крок 4: Емуляція підпису КЕП (Дія.Підпис або файл ключа) з красивою мікроанімацією верифікації.
   * Трекер статусу заяви («Отримана», «На розгляді», «Співбесіда призначена», «Оцінювання», «Схвалено / Відхилено»).

4. **Публічний реєстр та панель адміністратора (Registry & Admin Panel)**:
   * Пошук у реєстрі виданих сертифікатів за ПІБ або номером сертифіката (формат `СС ХХХХХХХХ/УУУУУУ-ZZ`).
   * Вбудована адмін-панель для перевірки надісланих заяв, зміни їхнього статусу та генерації офіційного сертифіката з автоматичним додаванням запису в реєстр.

### Manual Verification
1. Перевірка адаптивності інтерфейсу на мобільних пристроях та десктопах.
2. Проходження повного циклу тестування: відповіді на 24 питання, перевірка коректності розрахунку балів по функціях, генерація сертифіката.
3. Проходження майстра подачі заяви: заповнення полів, перевірка валідації освіти, завантаження документів, симуляція підпису КЕП.
4. Перевірка панелі адміністратора: схвалення заяви, автоматична генерація сертифіката у правильному форматі, перевірка оновлення публічного реєстру.
